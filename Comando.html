Configurar Repositorio Remoto

Con el comando git remote, podemos administrar y configurar repositorios remotos, en particular este comando nos muestra los repositorios remotos que tenemos configurados.

Subir confirmaciones

Para poder subir confirmaciones primero tenemos que agregar a nuestro repositorios el repositorio remoto con el que vamos a trabajar. Utilizamos git remote add (nombre) (url). El comando git remote -v listara los nombres de los repositorios y sus enlaces.

Configurar rama local con remota

  • Subiendo las confirmaciones:
  • git push –u origin main
  • git push --set-upstream origin main
  • Sin subir las confirmaciones:
  • git branch –u origin/main
  • git branch --set-upstream-to=origin/main

Git Branch

Con el comando git branch podemos conocer la rama en la que estamos trabajando.

  • Algunos comandos son:
  • git branch -M (nombre de la rama) para cambiar de rama
  • git branch -v camiamos el nombre del archivo

Git Checkout

Se utiliza para cambiar de una rama a otra, crear una y cambiarnos a esa y deshacer cambios de archivos.

  • Comando:
  • git checkout nombre (nombre de la rama) para cambiar el puntero a esa rama

Git Merge

La fusión es la forma que tiene Git de volver a unir un historial bifurcado

  • Comando:
  • git merge rama para fusionar el contenido de esa rama en la rama que estamos parados

Git Reset

Se usa para volver a versiones anteriores y limpiar el repositorio

  • Comandos:
  • git reset --hard deshacer todos los cambios en archivos modificados y preparados.
  • git reset HEAD archivo, deshacer el archivo preparado.
